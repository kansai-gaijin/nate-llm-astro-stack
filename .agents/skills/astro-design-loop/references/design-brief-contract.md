# Design brief contract

Return one implementation-oriented brief with these sections:

1. **Reference evidence**: inspected URLs, viewport sizes, capture states, blocked states, and dates.
2. **Design system**: colors, surfaces, borders, shadows, spacing rhythm, grid, breakpoints, and
   recurring component geometry.
3. **Typography**: observed reference fonts, closest Google Fonts replacement, weights, size/line
   height/letter spacing, language coverage, and substitution rationale.
4. **Page architecture**: every sitemap route and an ordered mapping from approved copy to reference
   patterns. Identify copy that does not fit naturally; do not omit it.
5. **Responsive behavior**: explicit desktop/tablet/mobile changes rather than generic “stack on
   mobile” guidance.
6. **Motion and state table**: loading, entrance, scroll, hover, focus, click, menu, carousel, and
   reduced-motion behavior with timing/easing when observable. Define continuity, reversal, and
   interruption behavior; abrupt movement is not acceptable.
7. **Media plan**: generated-image briefs, royalty-free video search requirements, Iconify names,
   crops/aspect ratios, and fallbacks.
8. **microCMS model**: only content that is genuinely dynamic, with endpoint type and field outline.
9. **Acceptance risks**: uncertainty, inaccessible evidence, content/reference mismatch, and the
   highest-risk fidelity areas.
10. **Creative elevation**: after baseline fidelity, identify a small set of coherent improvements
    to composition, storytelling, media, or motion that can raise the work toward award-worthy craft.
    State whether CSS/Alpine, GSAP, or Three.js is justified and why.

Use concrete values and selectors where evidence permits. Distinguish observation from inference.
